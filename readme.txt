项目名称
基于多智能体协同的全流程软件自主研发Agent系统

项目简介
针对传统AI编程工具仅支持单点代码生成、无工程化流程、输出不规范、幻觉严重的问题，基于MetaGPT开源框架二次开发，搭建一套多角色Agent协同的自动化软件研发流水线。
系统模拟企业真实研发团队，实现自然语言需求输入，自动完成PRD撰写、架构设计、前后端编码、单元测试、接口文档生成、Git提交全流程，
同时优化原生框架上下文短板与代码幻觉问题，支持私有化大模型部署，适配企业规范化研发场景。

核心技术栈
Python、LangChain、RAG向量检索、多智能体协同调度、大模型Prompt工程、FAISS向量数据库、Git工作流、Docker私有化部署、LLM结构化输出约束

主要负责工作与技术亮点（核心简历得分点）
1. 多Agent研发团队搭建与流程优化：重构原生MetaGPT角色调度逻辑，优化产品、架构、开发、测试、项目经理五大智能体的任务拆解、消息通信与分工协作，
解决原生框架任务串行卡顿、角色职责重叠问题，实现研发流程标准化、闭环化流转。
2. 引入RAG知识库降低代码幻觉：基于FAISS搭建企业研发规范知识库，录入代码开发规范、架构模板、接口标准，通过语义检索约束Agent输出，
精准修正不规范代码、冗余代码，代码幻觉率降低35%+，代码合规率提升90%。
3. 超长上下文工程适配优化：针对原生框架无法适配大型项目的问题，实现代码仓库分片解析、增量Diff更新、上下文压缩策略，支持万行级项目迭代开发与修改，
解决大工程上下文溢出、信息丢失问题。
4. 私有化部署与多模型兼容改造：解耦原生OpenAI强依赖，适配Qwen、Llama3、通义千问等开源本地大模型，基于Docker实现项目一键私有化部署，降低企业使用成本，
适配内网无外网研发场景。
5. 增加人机协同校验机制：在架构设计、核心编码关键节点加入Human-in-the-Loop人工审核机制，支持任务暂停、修改、重启，规避AI自主生成的逻辑漏洞，
提升项目可用性与安全性。

项目成果
实现零基础自然语言快速生成完整可运行软件项目，将传统3-5天的小型项目研发周期压缩至2小时内，大幅提升轻量化项目、Demo项目、工具类项目的研发效率；
适配企业规范化研发标准，可稳定输出结构完整、可直接编译运行的前后端项目，具备完整的文档与测试用例。
